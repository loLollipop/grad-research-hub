# 研途 Hub 产品优化计划

## 调研结论

研途 Hub 不应该成为 Zotero、Obsidian、电子实验记录本或项目管理软件的替代品。理工科研究生真正缺的是一个轻量工作台，把这些工具之间最耗时间的断点串起来：今天要做什么、文献读到哪里、实验是否有结论、结果能不能支撑论文、组会和材料有没有遗漏。

参考方向：

- Zotero 的强项是文献条目、集合、标签、附件和 Web API，同步应优先复用它。参考：[Zotero Web API](https://www.zotero.org/support/dev/web_api/v3/basics)。
- eLabFTW 这类 ELN 的强项是结构化实验、模板、状态、标签、导出和 API，但完整 ELN 对个人研究生偏重。参考：[eLabFTW Documentation](https://doc.elabftw.net/)。
- OSF 的强项是项目、组件、文件、Wiki、元数据和注册式研究流程，适合参考“项目容器”的思想。参考：[OSF API](https://developer.osf.io/)、[OSF Components](https://help.osf.io/article/622-utilize-components-for-detailed-project-management)。
- Obsidian 的强项是自由笔记、双链、标签和知识网络，研途 Hub 只需要吸收“连接”和“回顾”的价值。参考：[Obsidian internal links](https://obsidian.md/help/links)、[Obsidian tags](https://obsidian.md/help/tags)。

## 高需求功能

首批继续加强这些高频场景：

1. 今日工作台：把任务、实验、文献、笔记、轻行政和结果状态合并到第一屏，减少每天开工的判断成本。
2. Zotero 文献台：文献来源以同步为主，手动录入只做兜底；重点保留阅读状态、标签、笔记和待读队列。
3. 实验记录：用短模板记录目的、方法、观察、结论和下一步，不把界面做成参数仓库。
4. 项目推进：项目只承担阶段、里程碑和下一步，不堆过多字段。
5. 笔记连接：保留自由写作和 `[[双链]]`，未来把文献、实验、组会和写作草稿串起来。
6. 结果与写作：数据结果不追求完整 ML 平台，先做指标、复现实验状态、可写入论文的证据表。
7. 设置中心：AI Key、Zotero Key、访问密码等高频变动项放网页端，部署底层配置留给服务器。
8. 研究时间线：把文献、实验、结果和笔记按更新时间串起来，服务组会回顾、周报和论文材料追溯。

## 暂不做

- 多用户、课题组权限和审批流。
- PDF 文件托管和全文 RAG。
- 完整 LIMS、样品库存、仪器预约。
- 复杂知识图谱和自动论文写作。
- 替代 Git、DVC、MLflow、Zotero 或 Obsidian 的深层能力。

## UI 方向

目标是“舒服但不松散，现代但不浮夸”。界面应该像一个安静的科研驾驶舱，而不是传统后台。

- 第一屏优先显示行动建议和状态汇总，而不是大段说明。
- 页面使用柔和浅色底、少量深色焦点、低饱和点缀色，避免长时间使用疲劳。
- 卡片保持小圆角和清晰层级，减少厚重阴影。
- 表单默认收起到按钮或弹窗，只有笔记和实验正文这种沉浸式输入保持展开。
- 导航要让用户明确知道当前在哪，页面切换要轻快。

## 第一轮落地

- 重做 Dashboard 为“今日作战台”：
  - 今日重点：临近任务、最近实验、待读文献、行政提醒。
  - 研究闭环：文献、实验、结果、笔记四类状态一屏可见。
  - 下一步队列：任务和事务合并排序。
  - 复盘区：最近实验、最近文献、最近笔记。
- 调整全局壳和配色：
  - 更轻的背景纹理。
  - 侧边栏从“后台菜单”改成“研究工作流”。
  - 顶部快速捕捉更像命令条。
- 保持当前数据模型不变，避免引入迁移风险。

## 第二轮落地

- 重做文献页为“Zotero 阅读台”：
  - 顶部明确“同步为主、手动兜底”。
  - 展示待读、读中、已读和最近同步状态。
  - 左侧提供阅读节奏和常用集合，右侧保留搜索、状态筛选和文献卡片。
  - Zotero 未配置时引导到设置中心，而不是要求用户回服务器改配置。
- 重做实验页为“实验日志”：
  - 页面突出目的、观察、结论和下一步，弱化参数仓库感。
  - 保留模板思想：目的/方法/结果、消融、复现、问题排查。
  - 展示进行中、已完成、失败案例、已登记结果，鼓励保留负结果。
  - 编辑仍用弹窗，列表区域只显示可读日志、关联论文和结果证据。

## 第三轮落地

- 重做项目页为“课题路线图”：
  - 顶部只保留活跃课题、待推进任务、7 天内截止和实验连接四个信号。
  - 左侧新增“下一步队列”和“临近里程碑”，按截止与优先级帮助用户开工。
  - 右侧保留轻看板，但每张任务卡只展示标题、归属、截止、优先级和状态。
  - 项目、里程碑、任务的新增和编辑都改为弹窗，避免页面长期展开表单。
- 重做笔记页为“笔记工作室”：
  - 修复中文乱码，统一为中文界面。
  - 左侧保留搜索、分类和最近笔记，右侧使用更高的写作面板。
  - 笔记正文不使用弹窗，保持沉浸式长编辑区。
  - 预览和双链拆成标签页，双链页展示当前引用和已匹配笔记。

## 第四轮落地

- 重做成果页为“结果证据台”：
  - 页面目标从“管理数据”改为“留下能支撑结论的证据”。
  - 顶部突出关键结果、已复现、论文素材和进行中实验。
  - 左侧增加待补证据队列和记录原则，右侧展示最近指标图、关键结果、论文素材和数据集。
  - 结果和数据集编辑改为弹窗，避免折叠表单占据列表区域。
- 重做事务页为“轻事务时间线”：
  - 围绕组会、材料、报销、截止四类高频事项组织界面。
  - 顶部展示待处理、今天截止、7 天内事项和当前筛选结果。
  - 左侧显示当前最该处理事项和类型分布，右侧按时间线卡片展示事务。
  - 新增和编辑都走弹窗，列表只展示标题、状态、截止、地点和备注。
- 重做 AI 页为“科研助手”：
  - 不做万能聊天框，先提供组会提纲、实验复盘、阅读卡片和写作润色四类入口。
  - 保留 `/api/ai` 占位接口和设置中心配置，密钥不在浏览器明文展示。
  - 试验台增加预设提示词按钮，便于后续接真实模型时沿用。

## 第五轮落地

- 首页增加“空库开箱上手”状态：
  - 当数据库还没有任何论文、项目、任务、实验、笔记、结果或事务时，不再展示一堆 0 和空卡片。
  - 第一屏改为“10 分钟开箱”，引导用户先完成设置中心、Zotero 同步、第一条课题/任务三步。
  - 增加四个起步入口：设置中心、Zotero 阅读台、课题路线图、第一条记录。
  - 明确告诉用户暂时不用导入全部 PDF、迁移全部旧实验或配置不常用字段。
- 完成移动端视觉验收修复：
  - 移动端顶部导航改为更轻的横向工具条。
  - 快速捕捉在小屏压缩为短输入，减少首屏空间占用。
  - `.tmp/` 加入忽略规则，避免浏览器截图临时产物干扰 lint。

## 第六轮落地

- 文献页增加 Zotero 同步反馈：
  - 点击“同步 Zotero”后，成功会回到文献页并显示同步条数。
  - 如果 Zotero 返回 0 条可同步文献，会提示检查 Collection Key、同步数量和集合内容。
  - 同步失败会显示可理解的错误原因，并继续引导用户检查设置中心。
  - 不新增数据表，不暴露 API Key，仍然保持 Zotero 为文献源头。

## 第七轮落地

- 新增调研驱动优化计划：`docs/research-driven-optimization-plan.md`。
- 全局视觉从单一绿调调整为低疲劳纸面、靛蓝主色、青绿辅助色。
- 侧边栏改成更明确的科研工作流导航，当前页面选中态更清楚。
- 顶部快速捕捉改为更像命令入口，强调想法、待办和组会提醒。
- Dashboard 增加“先定下一步 -> 文献输入 -> 实验推进 -> 结果证据”的科研节奏条。
- README 和 Roadmap 增加调研计划入口，方便开源用户理解产品边界。

## 第八轮落地

- 设置中心增加 Zotero 连接测试：
  - 使用当前表单中的 API Key、Library ID、Library Type、Collection Key 和同步数量发起真实 Zotero API 探测。
  - 测试结果只显示可读摘要，不回显 API Key。
  - 如果输入框留空，则优先使用已经保存或环境变量中的 Key。
- 设置中心增加 AI 配置测试：
  - 检查 API Key、Base URL 和模型名，尝试访问模型列表探测端点。
  - 对 404、429 等常见中转网关/限流情况给出更像人话的提示。
  - 明确当前 AI 助手仍是 MVP 接口骨架，测试不等于已经接入真实聊天。
- 表单按钮调整为“保存”和“测试当前配置”并列，方便先试后存。

## 第九轮落地

- 文献页增加批量阅读状态更新：
  - 每张文献卡增加轻量复选框，不影响编辑、删除和单篇状态更新。
  - 列表顶部增加批量操作栏，可一次标记为待读、读中或已读。
  - 批量操作完成后回到当前筛选条件，并显示更新条数。
- 集合筛选更像 Zotero 阅读台：
  - 左侧增加“全部集合”入口和当前集合高亮。
  - 支持在保留关键词/状态筛选的情况下切换集合。
  - 顶部显示当前列表数量、筛选数量和一键清除筛选入口。

## 第十轮落地

- 实验表单改为模板感知：
  - 新增客户端实验表单组件，切换模板时会自动预填 Markdown 正文结构。
  - 已经手动写过正文后，切换模板不会覆盖用户内容。
  - 模板配置抽到共享文件，页面侧边栏和表单共用同一份文案。
- 失败实验增加复盘入口：
  - 标记为失败的实验卡片会显示“生成复盘任务”。
  - 复盘任务会自动创建为高优先级待办，并带上失败现象、可能原因、对照和下一次修改的检查项。
  - 如果实验关联项目且项目已有里程碑，任务会挂到最近里程碑；否则作为独立任务进入项目页队列。

## 第十一轮落地

- 实验页增加轻量筛选条：
  - 支持按关键词搜索标题、正文、标签、项目名和关联论文。
  - 支持按项目、状态和模板筛选。
  - 筛选后显示当前列表数量、已选项目、状态、模板，并提供一键清除筛选。
- 空状态根据是否筛选过显示不同文案，避免用户误以为数据丢了。

## 第十二轮落地

- 项目页增加任务筛选条：
  - 支持按关键词搜索任务、备注、标签、里程碑和项目。
  - 支持按项目、任务状态和优先级筛选。
  - 下一步队列和看板会跟随筛选结果，项目路线图仍保留全局视角。
  - 筛选摘要显示当前任务数量、已选项目、状态、优先级，并提供一键清除。
- 看板空列在筛选状态下显示“当前筛选下无任务”，避免误解为没有任务数据。

## 第十三轮落地

- 成果页增加证据筛选条：
  - 支持按关键词搜索结果标题、结论、指标、图表路径、关联实验和数据集。
  - 支持按复现状态、论文素材状态、实验和数据集筛选。
  - 筛选后顶部关键结果、已复现、论文素材等信号跟随当前范围更新。
  - 待补证据队列、关键结果列表和论文素材列表跟随筛选结果。
  - 筛选摘要显示当前结果数量、复现状态、素材状态、实验、数据集，并提供一键清除。
- 关键结果空状态根据是否筛选过切换文案，避免用户误以为结果丢失。

## 第十四轮落地

- 项目页增加“开工视图”：
  - 支持在任务看板上方一键切换全部、今天、本周。
  - 今天/本周默认只看未完成任务，避免已完成任务干扰当天开工。
  - 快捷视图会保留关键词、项目、状态和优先级筛选。
  - 筛选摘要增加当前快捷视图标签，并仍提供一键清除。
- 项目任务增加“转成实验”：
  - 未完成任务卡片和下一步队列可一键创建实验日志。
  - 新实验会自动关联任务所在项目，正文预填目的、方法、观察、结论和来源任务。
  - 原任务自动切到“进行中”，减少从课题推进到实验记录的重复操作。
  - 不新增数据库表，不引入新部署配置。

## 第十五轮落地

- 成果页增加“生成汇报清单”：
  - 可以把当前筛选出的结果证据一键整理成 Markdown 笔记。
  - 清单包含关联实验、数据集、核心指标、复现状态、图表路径、一句话结论和更新时间。
  - 生成后的笔记自动进入“组会”分类，方便继续改成组会提纲、论文图表清单或周报。
  - 没有结果时给出轻提示，不让用户点完以后毫无反馈。
  - 不做复杂导出系统，也不新增数据表，优先解决“组会前整理材料很烦”的高频痛点。

## 第十六轮落地

- Zotero 同步增加集合名称映射：
  - 同步文献时额外读取 Zotero collections，把条目里的 collection key 映射成集合名。
  - 如果集合列表读取失败，不阻塞文献同步，保留原 key 作为兜底。
  - 文献页“常用集合”会显示更接近 Zotero 的人类可读分类，减少用户看到一串 key 的困惑。
- 文献页最近同步状态改为全库维度：
  - 最近同步时间不再受当前搜索、阅读状态或集合筛选影响。
  - 避免筛选到某个子集时误显示“暂无同步”。

## 第十七轮落地

- 成果页增加“生成待补任务”：
  - 待补证据队列里的结果可以一键变成项目任务。
  - 关键结果卡片在复现未完成或素材未补齐时显示“生成待补任务”。
  - 任务会自动写入结果标题、关联项目/实验/数据集、核心指标、复现状态、图表路径和下一步检查项。
  - 如果结果关联的实验属于某个项目，会尽量挂到该项目最近里程碑；否则作为独立高优先级任务进入项目页。
  - 完成后跳转到项目页的高优先级待办视图，形成“证据缺口 -> 下一步任务”的闭环。

## 第十八轮落地

- 实验页增加“结果回填正文”：
  - 每条实验卡片的结果记录区显示最近结果，并提供“回填正文”入口。
  - 回填会把结果标题、数据集、核心指标、复现状态、图表路径和结论/下一步追加到实验正文末尾。
  - 原实验记录不会被覆盖；每条结果带隐藏标记，重复点击不会反复追加。
  - 如果实验仍处于进行中，回填后自动收口为完成，帮助用户把实验从“跑完了”推进到“有结论了”。
  - 回填后回到对应项目/状态的实验筛选视图，减少页面跳转成本。

## 第十九轮落地

- 项目页增加“今天高优先级”快捷入口：
  - 在开工视图中一键切到今天/逾期范围内的高优先级任务。
  - 保留当前关键词和项目筛选，但清掉任务状态筛选，避免已完成任务干扰当天开工。
  - 快捷入口直接服务“今天先做什么”，不增加新的配置项。
- 项目页增加任务批量状态更新：
  - 当前筛选列表里的任务卡片可以勾选后批量标为待办、进行中或完成。
  - 批量操作完成后回到原筛选视图，并显示处理条数和目标状态。
  - 未勾选任务时给出轻提示，避免用户点完没有反馈。
  - 不新增数据库表，不改变部署配置，继续保持低负担工作台。

## 第二十轮落地

- 首页和事务页增加“生成组会准备”：
  - 一键汇总本周高优先级/临近截止任务、未完成事务、最近实验、结果证据和待读文献。
  - 自动生成 Markdown 笔记并进入“组会”分类，用户继续在笔记页编辑，不引入新的汇报中心。
  - 草稿保留“本周最需要讲清楚”“下一步任务”“实验进展”“结果证据”“文献阅读”“事务提醒”“会后回填”等组会常用结构。
  - 这个功能参考了 ELN/OSF 的项目-记录-导出思想，但只保留个人研究生最常用的周会整理场景，避免做成重型协作系统。
  - 不新增数据库表、不新增部署配置，只把已有数据串成可用输出。

## 第二十一轮落地

- 组会草稿增加同周复用：
  - 生成草稿时写入隐藏周期标记，同一周再次点击会直接打开已有笔记，不再重复创建一堆相似草稿。
  - 首页和事务页会显示“本周组会草稿已经准备好”、覆盖日期和最近更新时间。
  - 已有草稿时主按钮改为“继续编辑草稿”，没有草稿时才显示“生成组会准备”。
  - 周期计算统一放到共享工具中，避免页面和 server action 对“本周”的理解不一致。
  - 仍然不新增数据库表，不增加用户配置，把重复劳动藏到系统里处理。

## 第二十二轮落地

- 笔记页增加“待办清单拆成任务”：
  - 识别当前笔记正文中的 `- [ ]` 未完成清单，一键创建为项目任务。
  - 创建后跳转到项目页待办视图，并显示创建数量，用户后续再把任务挂到对应里程碑。
  - 原笔记会写入隐藏同步标记，重复点击不会把同一条清单反复创建成任务。
  - 按钮只在有可同步清单时可用，避免用户点完没有反馈。
  - 不把笔记页变成任务管理页，只补“组会/阅读/实验笔记 -> 下一步任务”的高频闭环。

## 第二十三轮落地

- 项目页增加“快速挂到里程碑”：
  - 独立任务卡片和下一步队列在有里程碑时显示紧凑挂载入口。
  - 用户可以直接选择里程碑并保存，不需要打开完整编辑弹窗。
  - 挂载后回到原筛选视图，并给出成功或失败提示。
  - 这个入口主要承接“笔记清单拆成任务”后的下一步整理，把临时任务快速放回课题路线图。
  - 不新增数据库表、不新增配置项，只减少一次高频操作的点击成本。

## 第二十四轮落地

- 首页增加“研究时间线”：
  - 将最近文献、实验、结果和笔记合并成按更新时间排序的时间线。
  - 替换分散的“最近实验 / 文献与笔记”板块，减少首页重复信息和视觉疲劳。
  - 每条记录显示类型、标题、上下文、更新时间，并直接跳转到对应模块。
  - 这对应 ELN/OSF/Obsidian 共同强调的“记录可追溯”和“材料可回顾”，但不新增重型协作或知识图谱功能。
  - 不新增数据库表、不引入配置项，继续复用已有数据。

## 第二十五轮落地

- 首页增加“收口提醒”：
  - 自动识别逾期任务、久未更新的进行中任务、久未收口实验、待补复现/论文素材的结果、长时间未处理文献。
  - 每次最多显示 4 条，避免把首页变成焦虑清单。
  - 每条只显示原因和下一步动作，并跳转到已有模块处理。
  - 没有风险时显示“状态稳定”，给用户一个明确反馈。
  - 不新增数据库表、不新增配置项，继续围绕“少想一步、少漏一件事”的研究生日常需求优化。

## 第二十六轮落地

- 顶部快速捕捉增加轻前缀分流：
  - 没有前缀时仍默认创建 Inbox 笔记，保留原来的随手记行为。
  - 支持 `任务：`、`实验：`、`文献：`、`事务：` 等前缀，直接创建对应记录。
  - 快速创建的任务默认进入独立待办，后续可在项目页快速挂到里程碑。
  - 快速创建的实验会预填目的、方法、观察、结论/下一步结构。
  - 不新增弹窗、不新增配置，只减少“想到一件事 -> 找页面 -> 打开表单”的操作成本。

## 第二十七轮落地

- 文献页增加“一键阅读笔记”：
  - 每篇文献可直接生成结构化阅读卡片，并跳转到笔记工作室继续编辑。
  - 阅读卡片包含文献信息、一句话问题、方法抓手、关键结果、可复用点、疑问和原始摘要/备注。
  - 同一篇文献重复点击会打开已有笔记，不会重复生成一堆卡片。
  - 未读文献生成阅读笔记后自动切到“读中”，贴合真实阅读流程。
  - 不做 PDF 托管、全文 RAG 或 Zotero 替代，只补“文献条目 -> 可写作笔记”的高频闭环。

## 第二十八轮落地

- 实验页增加“一键复盘笔记”：
  - 每条实验可直接生成结构化复盘笔记，并跳转到笔记工作室继续编辑。
  - 复盘笔记自动带入项目、状态、关联论文、结果指标、原始实验记录和下一步清单。
  - 同一条实验重复点击会打开已有复盘笔记，不重复创建。
  - 与失败实验的“生成复盘任务”互补：任务负责行动，笔记负责沉淀和组会/周报回顾。
  - 不新增数据库表，不做重型 ELN，只补“实验记录 -> 可回顾复盘”的高频闭环。

## 第二十九轮落地：研究指挥台与低疲劳 UI

- 调研结论：继续参考 Zotero、eLabFTW、OSF、Obsidian 和研究数据管理实践，但只吸收高频价值点：文献队列、实验记录、课题推进、结果证据、笔记连接和组会/周报整理。
- 首页策略：把第一屏改成“今日指挥台”，先给下一步和 01/02/03 行动栈，再给研究流信号，减少统计卡片堆叠带来的后台感。
- 视觉策略：全局背景从明显网格纹理改成柔和层次纸面，主色转向冷静钴蓝和青绿，降低长时间使用疲劳。
- 侧栏策略：侧栏顶部从普通模块提示改成“文献 -> 实验 -> 成果”的科研流程提示，让用户知道平台的核心是闭环，而不是配置中心。
- 边界：不新增数据库表、不新增配置项、不做重型知识图谱或 ELN，只把已有数据组织成更好理解、更愿意每天打开的工作台。

## 第三十轮落地：Zotero 阅读栈

- 文献页策略：继续坚持 Zotero 同步优先，研途 Hub 只负责阅读状态、阅读队列、筛选和阅读笔记沉淀。
- 第一屏调整：把文献页 hero 改成和首页一致的低疲劳指挥台视觉，并加入 01/02/03 今日阅读栈。
- 快捷视图：侧栏新增全部文献、待读队列、正在读、已读回顾入口，保留当前关键词和集合筛选。
- 卡片调整：每篇文献显示“开始阅读 / 继续阅读 / 回顾笔记”下一步提示，让用户不用在状态字段里猜下一步。
- 边界：不新增 PDF 托管、全文 RAG、复杂引用网络或新配置项，优先解决“今天先读哪几篇”的高频痛点。

## 第三十一轮落地：实验日志栈

- 实验页策略：继续保留轻量 ELN 思路，重点是模板、状态、观察、结论、结果回填和失败复盘，不做参数仓库。
- 第一屏调整：把实验页 hero 改成和首页一致的低疲劳指挥台视觉，并加入 01/02/03 今日实验栈。
- 快捷视图：侧栏新增全部实验、继续观察、已完成、失败复盘入口，保留当前关键词、项目和模板筛选。
- 卡片调整：每条实验显示“继续观察 / 补结果 / 沉淀结论 / 复盘失败”下一步提示，减少用户在状态和正文之间来回判断。
- 边界：不新增 LIMS、样品库存、仪器预约或新配置项，优先解决“实验做完但复盘和结果证据没有收口”的高频痛点。

## 第三十二轮落地：课题推进栈

- 项目页策略：不照搬 Jira/Notion，也不做甘特图；只回答“今天课题下一步是什么、哪几件事要收口”。
- 第一屏调整：把项目页 hero 改成和首页一致的低疲劳指挥台视觉，并加入 01/02/03 今日推进栈。
- 快捷视图：侧栏新增全部任务、今天/逾期、高优先级、进行中、本周收口入口，保留当前关键词和项目筛选。
- 卡片调整：任务卡显示“开始推进 / 继续推进 / 优先处理 / 先补逾期 / 补里程碑 / 已收口”下一步提示。
- 边界：不新增数据库表、不做多人协作、文件仓库或复杂权限，优先解决“项目很多但不知道先推进哪一步”的高频痛点。

## 第三十三轮落地：结果证据栈

- 成果页策略：不做重型数据仓库，也不把实验参数再抄一遍；只回答“这条结果能不能支撑结论、还缺什么证据”。
- 第一屏调整：把成果页 hero 改成和首页一致的低疲劳指挥台视觉，并加入 01/02/03 今日证据栈。
- 快捷视图：侧栏新增全部结果、待判断、待复现、复现中、已复现和可写入入口，保留当前关键词、实验和数据集筛选。
- 卡片调整：结果卡显示“先判定复现 / 补复现实验 / 继续复现 / 补图表路径 / 标成写作素材 / 已成证据”下一步提示。
- 边界：不新增数据库表、不做文件仓库、样本管理或复杂 FAIR 元数据，只把关键结果推进到可复现、可汇报、可写作的状态。

## 后续路线

1. 文献页：继续增加 Zotero 增量同步版本提示和更细的同步错误定位。
2. 实验页：继续增加按失败原因归类和结果回填后的重复提示。
3. 项目页：继续补重复任务提醒、批量删除前确认和更清晰的逾期分组。
4. 成果页：继续补图表路径可达性检查、结果重复提醒和按项目汇总证据清单。
5. 设置页：继续补更细的错误定位，例如 Zotero 权限不足、Collection Key 不存在、AI 网关不兼容。
6. 组会笔记：继续补草稿内容的局部刷新和任务挂载到里程碑的轻量入口。
7. AI 页：接入真实模型调用前，先补服务器端速率限制和错误状态。
8. 首页：真实使用一周后，根据数据密度继续精简模块顺序。
