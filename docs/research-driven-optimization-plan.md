# 研途 Hub 调研驱动优化计划

## 调研摘要

研途 Hub 的目标不是替代 Zotero、Obsidian、eLabFTW、OSF 或通用项目管理软件，而是做一个理工科研究生每天愿意打开的轻量工作台。调研后得到的核心判断是：研究生最需要的不是更多字段，而是更少的决策、更顺的切换、更清楚的下一步，以及把文献、实验、结果、写作和轻事务之间的断点接起来。

## 参考工具与可借鉴能力

- Zotero：Web API v3 支持按库、集合、标签、搜索等方式读取文献条目，并建议生产代码指定 API 版本；研途 Hub 应继续把 Zotero 当作文献源头，而不是要求用户重复录入文献。参考：<https://www.zotero.org/support/dev/web_api/v3/basics>
- eLabFTW：实验条目是 ELN 核心，模板可节省重复实验创建时间，实验记录包含标题、分类、状态、标签、正文、步骤、链接和附件等；研途 Hub 只吸收“模板 + 状态 + 正文”的个人轻量部分，不做完整 ELN。参考：<https://doc.elabftw.net/docs/usage/user-guide/experiments/>、<https://doc.elabftw.net/docs/usage/user-guide/templates/>
- OSF：项目可用 components 按研究阶段、材料类型、数据、分析、手稿等拆分，并用 wiki 记录过程；研途 Hub 应借鉴“项目容器 + 阶段化推进”，但不做协作者权限、文件仓库和 DOI 流程。参考：<https://help.osf.io/article/622-utilize-components-for-detailed-project-management>
- Obsidian：内部链接和 backlinks 能把笔记变成知识网络；研途 Hub 保留 `[[双链]]` 识别、最近笔记和写作工作区即可，不急着做完整知识图谱。参考：<https://obsidian.md/help/links>、<https://obsidian.md/help/plugins/backlinks>
- 研究数据管理研究：多项 RDM 研究指出研究生在数据组织、文档化、保存、共享、数据管理计划等方面存在能力缺口；研途 Hub 的“结果证据台”应强调可复盘、可写入论文、可追溯，而不是变成复杂数据平台。参考：<https://pmc.ncbi.nlm.nih.gov/articles/PMC9949669/>、<https://journals.library.ualberta.ca/istl/index.php/istl/article/download/2678/2709>

## 高需求功能优先级

1. 今日工作台：打开后先知道“下一步做什么”，将任务、实验、文献、事务截止合并排序。
2. Zotero 阅读台：同步优先，手动补录只是兜底；强化阅读状态、标签、集合筛选、批量状态更新。
3. 实验日志：让用户快速写清目的、方法、观察、结论、下一步；模板选择要比字段堆叠更重要。
4. 课题路线图：项目页面重点保留阶段、里程碑、任务和最近推进，不展示过量元信息。
5. 笔记工作室：保留沉浸式编辑，不把写笔记塞进弹窗；强化最近笔记、双链、预览和引用关系。
6. 结果证据台：围绕“这条结果能不能支撑论文”组织指标、数据集、复现状态和素材路径。
7. 轻事务时间线：组会、材料、报销、截止提醒按时间线处理，避免散落在微信、备忘录和表格里。
8. 设置中心：AI Key、Base URL、Zotero Key、访问密码放网页端改；数据库、端口、加密密钥留给部署配置。

## UI 设计原则

- 第一屏显示行动建议，不显示说明书。
- 低饱和纸面背景 + 靛蓝主色 + 青绿/暖砂辅助色，避免长时间使用视觉疲劳。
- 导航是“研究流程”，不是后台菜单；当前页面必须明确高亮。
- 列表、卡片和表单保持紧凑，但留足阅读空间。
- 表单默认收起到按钮或弹窗，笔记编辑器和实验正文除外。
- 首页用“下一步 -> 文献输入 -> 实验推进 -> 结果证据”的科研节奏条解释产品，而不是用营销文案解释功能。

## 当前落地轮次

- 全局视觉 token 调整为低疲劳纸面/靛蓝/青绿体系。
- 侧边栏增加轻量研究流程提示，导航选中态更清楚。
- 顶部快速捕捉改成更像命令入口，适合记录想法、待办和组会提醒。
- Dashboard 增加“科研节奏条”，将下一步、文献、实验、证据串成每日工作流。
- 保持数据库结构不变，避免为了 UI 改造引入迁移风险。

## 下一轮建议

1. 文献台：增加集合快捷筛选、批量改阅读状态、最近同步版本/错误细节。
2. 实验日志：模板选择后动态预填正文，失败实验一键生成复盘任务。
3. 设置中心：增加 Zotero 连接测试、AI 连接测试和更清晰的 ready/not ready 反馈。
4. 结果证据台：增加“可写入论文”证据清单和按项目/实验过滤。
5. 视觉验证：用桌面与移动端截图审查页面拥挤、空白、文字溢出和选中态一致性。
